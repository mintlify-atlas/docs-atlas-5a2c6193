Beyond submitting icons, you can contribute to the repository itself. Anyone is welcome to help with documentation improvements, website enhancements, bug fixes, and general maintenance.

## How to contribute

<Steps>
  <Step title="Fork the repository">
    Fork [homarr-labs/dashboard-icons](https://github.com/homarr-labs/dashboard-icons) on GitHub to create your own copy of the repository.
  </Step>
  <Step title="Make your changes">
    Work on your contribution locally. Common areas include:

    - Documentation improvements
    - Website enhancements
    - Bug fixes
    - Repository maintenance and tooling
  </Step>
  <Step title="Write semantic commit messages">
    Follow the `<type>(scope): description` format for all commits:

    ```text
    feat(icons): add nextcloud-calendar
    fix(website): correct icon preview
    docs(readme): update installation instructions
    ```

    Common types: `feat`, `fix`, `docs`, `chore`, `refactor`.
  </Step>
  <Step title="Submit a pull request">
    Open a pull request against the main branch. Reference any related issues in the PR description so the team can link your contribution to existing requests.
  </Step>
</Steps>

## What to contribute

<CardGroup cols={2}>
  <Card title="Documentation" icon="book">
    Fix inaccuracies, improve clarity, or add missing information to guides and reference pages.
  </Card>
  <Card title="Website enhancements" icon="globe">
    Improve the dashboardicons.com interface, fix display issues, or add useful features.
  </Card>
  <Card title="Bug fixes" icon="wrench">
    Report or fix bugs in scripts, workflows, or the website.
  </Card>
  <Card title="Repository maintenance" icon="gear">
    Help keep the repository organized — CI improvements, dependency updates, and tooling.
  </Card>
</CardGroup>
